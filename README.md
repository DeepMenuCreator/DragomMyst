# MystPlugin

Плагин "мистических сундуков" (Myst) для Paper 1.21.4.

## Установка / автосборка через GitHub

1. Создай новый репозиторий на GitHub и залей туда всё содержимое этой папки
   (сохраняя структуру, включая `.github/workflows/build.yml`).
2. Зайди во вкладку **Actions** репозитория — сборка запустится автоматически
   при пуше в ветку `main`/`master`, либо запусти вручную кнопкой **Run workflow**.
3. После завершения сборки открой завершённый workflow-run → раздел **Artifacts** →
   скачай `MystPlugin.zip`, внутри будет `MystPlugin.jar`.
4. Если хочешь релиз с готовым jar на странице Releases — запушь git-тег,
   например: `git tag v1.0.0 && git push origin v1.0.0`.
5. Закинь `MystPlugin.jar` в папку `plugins` своего Paper 1.21.4 сервера и перезапусти сервер.

## Команды

```
/myst delay                              - таймер ближайшего миста (открытие/закрытие)
/myst pos                                - координаты ближайшего миста
/myst menu <имя>                         - GUI-меню настройки миста
/myst reload                             - перезагрузить config.yml
/myst preview <имя>                      - превью возможного лута
/myst security <имя> <on/off>            - приват зоны при спавне
/myst kd <имя> <on/off>                  - таймер-уведомление при входе в зону
/myst use <имя> <on/off>                 - разрешить/запретить использование в зоне
/myst inventory <имя>                    - GUI-редактор лута
/myst create <имя>                       - создать новый тип миста
/myst editholo <имя> <текст>             - текст голограммы имени (поддержка & и &#RRGGBB)
/myst spawn <имя>                        - заспавнить мист под ногами (доп. команда)
```

Права: `myst.admin` (по умолчанию — оператор) на все настроечные подкоманды,
`myst.bypass` — игнорировать приваты зон Myst.

## Логика (как реализованы условия ТЗ)

1. **Таймер над мистом** — голограмма (TextDisplay) показывает обратный отсчёт до
   открытия, затем до закрытия, а после закрытия — только имя миста.
2. **Нельзя заспавнить мист над мистом** — при спавне проверяется расстояние до всех
   активных мистов (`settings.no-double-spawn-radius`, по умолчанию 15 блоков).
3. **Исчезновение голограммы вместе с мистом** — при деспавне удаляются и блок,
   и TextDisplay-голограмма.
4. **Вместо лута — "Сокровище"** — все 54 слота сундука заполняются предметом-заглушкой
   (по умолчанию Phantom Membrane, настраивается), помеченным через PersistentDataContainer.
   При попытке забрать такой предмет плагин перехватывает клик, удаляет заглушку и выдаёт
   игроку случайный предмет по таблице лута (вес/мин/макс количество).
5. **Кулдаун 1 секунда между предметами** — хранится per-player, per-myst
   (`settings.pickup-cooldown-ms`).
6. **Интерфейс как у эндер-сундука, но двойной сундук** — физический блок — Ender Chest,
   но при взаимодействии открывается кастомный инвентарь на 54 слота (визуально как
   двойной сундук) вместо ванильного личного эндер-хранилища.
7. **Нельзя открыть заранее** — пока мист в состоянии `WAITING_OPEN`, клик по блоку
   отменяется, инвентарь не открывается, игроку показывается оставшееся время.

## Структура конфигурации

`config.yml` → `myst-types.<id>` — шаблон (создаётся `/myst create`, редактируется
через `/myst menu`). `plugins/MystPlugin/active-mysts.yml` — список активных
заспавненных мистов (переживает рестарт сервера).

## Требования

- Paper 1.21.4 (или совместимый форк на этой версии API)
- Java 21+
